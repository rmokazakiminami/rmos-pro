# RMOS Pro 変更履歴

機能を追加・修正するたびに、ここに1行〜数行で記録する。
「いつから」「何が」変わったかが分かれば、不具合が起きたときに原因を探す時間が大幅に減る。

## 書き方のルール
- 新しい変更を一番上に追加する
- 日付は西暦/月/日
- 「追加」「修正」「削除」のどれかを最初に書く
- index.html と GAS（Apps Script）のどちらを変更したかを書く

## 2026-07-15

- **追加**（index.html）：公開エリアのFAQに「届出・申請の早見表」カテゴリを新規追加（アップロードされた「届出・申請ガイドブック」をQ&A形式に変換）。全14問、各回答に該当する規約・細則条文へのジャンプリンク付き。一覧の一番上に表示される
  - 参考：規約・細則の条文番号は令和8年7月19日施行の改正規約と既に一致しているため、総会後の番号修正は不要（ガイドブック作成時点で新番号を使用済み）
- **追加**（GAS）：意見箱で「個別」回答を公開したときの通知を追加。①誰宛かを明かさない汎用文面（「意見箱に個別回答が届いています」）でプッシュ通知を全体に送信、②連絡先にメールアドレスが登録されていれば、質問と回答をそのまま本人へメール送信（電話番号のみの場合はメール送信をスキップ）

## 2026-07-14

- **追加**（index.html / GAS）：意見箱の投稿フォームで、回答方法に「個別回答・電話・書面・訪問」を選んだ場合、部屋番号かメール／電話番号のどちらかの入力を必須化
- **修正**（GAS）：`publishAnswer` が I列（公開／個別）に書き込めなかった不具合を修正。原因はスプレッドシートのデータ入力規則が「個別」という値を許可していなかったため。書き込み直前に `clearDataValidations()` で規則を解除する形に変更
- **修正**（GAS）：意見箱・個別回答・システム確認パネルの日時表示が最大16時間ずれる不具合を修正。原因はスプレッドシートが日時の文字列を自動で日付型セルに変換し、ファイルのタイムゾーン設定とのズレが生じていたため。保存時に「書式なしテキスト」で固定する形に変更
- **修正**（index.html）：意見箱の回答送信ボタンが `no-cors` 通信を使っていたため、サーバー側で失敗していても画面上は「成功」と表示されてしまう不具合を修正。`cors` 通信に変更し、実際の成否を画面に表示するように変更
- **追加**（GAS）：`getMyAnswers`（個別回答の取得）・`getSystemNotices`（システムテスト投稿の確認）を実装。あわせて号室の全角・半角ゆれを吸収する `normalizeRoom_` を追加
- **追加**（index.html / GAS）：緊急プッシュ通知パネルからの送信を、Web PushとLINE公式アカウント（@118ketww）に同時配信するよう変更（GAS経由に統一し、チャネルアクセストークンをブラウザ側に置かないようにするため）
- **追加**（index.html / GAS）：通常のお知らせ投稿にも「LINEにも配信する」チェックボックスを追加（緊急・重要なときだけ管理者が都度選ぶ運用、無料枠を消費するためデフォルトはOFF）
- **追加**（GAS）：岡崎市防災メール（mail@city.okazakibosai.jp）の自動取込み機能（`checkBosaiMail`）。重大なもの（避難指示・特別警報等）のみWeb Push・LINE配信、それ以外はお知らせに記録のみ
- **追加**（index.html）：「システム（管理者テスト用）」配信先。選択すると投稿ログにのみ記録され、実際の配信は一切行われない（バグ修正時の動作確認用）
- **修正**（index.html）：お知らせの添付リンクを複数対応に変更（カンマ区切りで複数URL、2件以上は「📎資料1」「📎資料2」…とボタン表示）
- **修正**（index.html）：CSV解析処理が、本文が複数行のお知らせで列がズレる不具合を修正
- **修正**（index.html）：お知らせの「エリアを超えて配信される」不具合の一部原因だった、カレンダー予定の内容表示（直近の予定欄）を廃止し、カレンダーへのリンクのみに変更
